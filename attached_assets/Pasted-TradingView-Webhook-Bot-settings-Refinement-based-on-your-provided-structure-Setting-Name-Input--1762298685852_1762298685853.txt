Trade Execution & Order Management
Setting Name	Input Type	Interpretation & Purpose
Order Type Default	Radio Button (Market, Limit, Post Only Limit)	Defines the type of order the bot should use by default when receiving a signal, if the signal doesn't specify one.
Max Open Positions	Numeric (Integer)	Limits the total number of simultaneous open trades/positions the bot can manage across all tickers.
Minimum Position Size	Numeric (Value or Percentage)	Rejects signals if the calculated position size (after dynamic sizing) is below the minimum allowed by the exchange or user's preference, preventing dust trades.
Take Profit Percentage	Numeric (Percentage)	A default or mandatory setting to automatically place a Take Profit order at a specified percentage gain from the entry price, if the signal doesn't include one.
Stop Loss Percentage	Numeric (Percentage)	A default or mandatory setting to automatically place a Stop Loss order at a specified percentage loss from the entry price, if the signal doesn't include one.



Risk & Strategy Controls

Max Position Leverage	Numeric (e.g., $1\text{x}$ to $10\text{x}$)	Caps the maximum leverage the bot can use, overriding a higher leverage specified by the signal for risk control.

Ignore Tickers List	Text Area (Comma-separated list)	A blacklist of specific assets (e.g., BTC/USDT) that the bot should ignore, even if a signal is received.

Advanced Performance & Strategy Settings
Setting Name	Input Type	Interpretation & Purpose
Partial Fill Handling	Radio Button (Cancel Remainder, Wait for Fill)	Determines the bot's action if a limit order is only partially filled (e.g., $50\%$ of the size). Cancel Remainder places a new order for the rest; Wait for Fill keeps the order active.
Cooldown Period (per Ticker)	Seconds	After an exit signal for a specific ticker (e.g., BTC/USDT) is processed, the bot ignores any new entry signals for that ticker for this duration. Purpose: Prevents immediate re-entry on whipsaws.
Trailing Stop Activation	Percentage	The profit threshold (e.g., $+2\%$) after which the bot automatically converts the fixed Stop Loss (if present) into a Trailing Stop Loss.
Max Drawdown (Account Level)	Percentage	A critical Global Kill Switch. If the total floating or realized loss on the entire portfolio hits this percentage (e.g., $-20\%$), the bot immediately closes all open positions and pauses all trading.

Exchange Time Offset	Numeric (Milliseconds or Seconds)	Accounts for potential clock drift between the bot's server and the exchange's server. Crucial for signed API requests to prevent authentication failures.

Cost & Fee Optimization
Setting Name	Input Type	Interpretation & Purpose
Max Trade Fee Percentage	Percentage	A strict limit on the maximum accepted trading fee (taker or maker). If the bot calculates that the effective fee for an order will exceed this limit (e.g., during high-volume periods or for specific assets), the trade is rejected.
Minimum Profit Target ($)	Currency Value	Rejects trades where the calculated potential profit (based on signal or default TP setting) is less than the expected combined round-trip fees. Purpose: Ensures trades are financially viable and not just fee generators.
